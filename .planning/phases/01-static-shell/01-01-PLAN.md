## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| file:// → browser renderer | The HTML file is loaded from local disk; browser parses and renders it. No server, no network in Phase 1. |
| Author → HTML content | The developer writes the static HTML; no user input, no external data ingestion in this phase. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | hooch-snapshot.html (local file) | accept | File lives on the developer's own machine. No write access from browser context. Tampering would require local filesystem access, which is outside the browser threat model for a file:// page. |
| T-01-02 | Information Disclosure | Static HTML content | accept | Phase 1 contains no sensitive data — only placeholder strings and hardcoded station IDs that are public. No credentials, no tokens, no PII. |
| T-01-03 | Spoofing | None | accept | No authentication or identity concept exists in Phase 1. Not applicable. |
| T-01-04 | Repudiation | None | accept | No actions or transactions in Phase 1. Not applicable. |
| T-01-05 | Denial of Service | Browser parsing | accept | Self-contained file; no fetch calls, no loops, no timers. Cannot be DoS'd from a remote attacker. |
| T-01-06 | Elevation of Privilege | None | accept | No privileges exist in Phase 1. Not applicable. |
| T-01-07 | Injection / XSS (future risk) | Static HTML — all content is hardcoded strings | accept (Phase 1) / mitigate (Phase 2) | Phase 1 has no dynamic content — all strings are author-controlled literals. XSS is not possible when there is no data ingestion. **Phase 2 must sanitize USGS API response data before inserting into the DOM** (use textContent, not innerHTML). Flag this as a Phase 2 constraint. |
| T-01-SC | Tampering | npm/pip/cargo installs | not applicable | Phase 1 has zero package-manager installs. No registry interaction. slopcheck not needed. |

## HIGH Severity Findings

None in Phase 1. The file is fully static with no network requests, no user input, and no dynamic content. The XSS risk (T-01-07) is deferred to Phase 2 where it MUST be mitigated by using `textContent` instead of `innerHTML` when injecting USGS data.
